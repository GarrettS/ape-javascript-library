Overview

APE JavaScript Library

  • source: APE.js
  • author: Garrett Smith

    Released under Academic Free Licence 3.0. The general idea: high degree of abstraction and modularity with minimal architecture and framework bloat.

    Files under the DOM package, for example, deal with all sorts of various browser differences, but these are categorized into groups of activity-related objects.

    -f.js:

    Files that are in lower-case and end in "-f" example: "classname-f.js" contain functions that are added to the package of the folder they're contained in. This is keeps the global namespace clean, but doesn't add an extra object. By not adding an extra object, the overhead of an extra runtime lookup is avoided.

    -a.js

    Aspect files. These files are meant for cross-cutting (borrowing methods).

    Files with an UpperCase name can be thought of as Modules or Classes. These files contain an object with the name of the file, and usually nothing else, unless something else is very tightly coupled to it.

APE.addBefore(o, funcName, advice)
Wraps o's method with advice. If the advice returns false, o's method won't fire.
parameters
Object o the object that has the method.
string funcName o's named method.
Function advice to wrap. If returning false, o's method won't fire; If returning an array, the array is passed to apply to o's method call. Otherwise, just call o with the original arguments it was called with (default).
