Class String

Defined in APE.js

Some simple additions to the String object.

  • author: Garrett Smith

    Released under Academic Free Licence 3.0. The general idea: high degree of abstraction and modularity with minimal architecture and framework bloat.

    The APE.dom package, for example, deals with all sorts of various browser implementation differences, but these are categorized into groups of activity-related objects.

    -f.js:

    Files that are in lower-case and end in "-f" example: "classname-f.js" contain functions that are added to the package of the folder they're contained in. This is keeps the global namespace clean, but doesn't add an extra object. By not adding an extra object, the overhead of an extra runtime lookup is avoided. Functions are first-class objects in JavaScript, there really is no need for an extra object to hold functions as "static methods" in a package.

    -a.js

    Aspect files. These files are meant for cross-cutting (borrowing methods).

    Files with an UpperCase name can be thought of as Modules or Classes. These files contain an object with the name of the file, and usually nothing else, unless something else is very tightly coupled to it.
